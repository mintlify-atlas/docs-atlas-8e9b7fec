Susee is a TypeScript-first bundler built specifically for library package authors. It compiles and bundles your TypeScript source into dual ESM/CommonJS outputs, emits `.d.mts`/`.d.cts` declaration files, generates sourcemaps, and can automatically update your `package.json` fields — all from a single config file or one CLI command.

## Two ways to use Susee

Susee can be used as a CLI tool or as a Node.js library — pick the approach that fits your workflow.

<Tabs>
  <Tab title="CLI">
    Initialize a config file, then build:

    ```bash
    npx susee init
    npx susee
    ```

    Or build a single entry directly without a config file:

    ```bash
    npx susee build src/index.ts --format esm --outdir dist
    ```
  </Tab>
  <Tab title="Programmatic API">
    Import `build` and pass a config object directly from your build script:

    ```typescript
    import { build, type SuSeeConfig } from "susee";

    const config: SuSeeConfig = {
      entryPoints: [
        {
          entry: "src/index.ts",
          exportPath: ".",
          format: ["esm", "commonjs"],
        },
      ],
      outDir: "dist",
    };

    await build(config);
    ```
  </Tab>
</Tabs>

## Key features

<CardGroup cols={2}>
  <Card title="Dual format output" icon="files">
    Produce both ESM (`.mjs`) and CommonJS (`.cjs`) from a single source, with matching sourcemaps and declaration files.
  </Card>
  <Card title="TypeScript declarations" icon="file-code">
    Automatically emits `.d.mts` and `.d.cts` type declaration files alongside compiled output.
  </Card>
  <Card title="Post-process plugins" icon="plug">
    Extend the build pipeline with custom post-process plugins, including the built-in terser minification plugin.
  </Card>
  <Card title="package.json updates" icon="file">
    Optionally let Susee update `main`, `module`, and `types` fields in your `package.json` based on build output.
  </Card>
</CardGroup>

## Get started in four steps

<Steps>
  <Step title="Install Susee">
    Add Susee as a dev dependency: `npm install -D susee`
  </Step>
  <Step title="Initialize your config">
    Run `npx susee init` to create a `susee.config.ts` template at your project root.
  </Step>
  <Step title="Configure entry points">
    Edit `susee.config.ts` to specify your entry files, export paths, and output formats.
  </Step>
  <Step title="Build your library">
    Run `npx susee` to compile and bundle your library into the `dist/` directory.
  </Step>
</Steps>
