Susee is a TypeScript-first bundler designed specifically for library package authors. It takes your TypeScript source, compiles it, and produces dual ESM and CommonJS outputs — complete with sourcemaps, declaration files, and optional `package.json` field updates — from a single config file or one CLI command.

## Why Susee exists

Publishing a TypeScript library that works correctly for both ESM and CommonJS consumers requires emitting multiple formats, keeping declaration files in sync, and managing `package.json` fields like `main`, `module`, and `types`. Doing this manually with separate tools is repetitive and error-prone. Susee bundles all of that into one tool with a minimal config surface.

## Two ways to use Susee

### CLI

The `susee` binary covers three commands:

| Command | Purpose |
| --- | --- |
| `susee` | Build using `susee.config.ts` (or `.js` / `.mjs`) found in your project root |
| `susee init` | Generate a config template at your project root |
| `susee build <entry> [options]` | Build a single entry file directly from CLI arguments |

```bash
# Generate config, then build
npx susee init
npx susee

# One-off build without a config file
npx susee build src/index.ts --outdir dist --format esm
```

### Programmatic API

Import `build` from `susee` and pass a `SuSeeConfig` object directly. Useful when you want to drive builds from a custom script or integrate Susee into a larger pipeline.

```typescript
import { build, type SuSeeConfig } from "susee";

const config: SuSeeConfig = {
  entryPoints: [
    {
      entry: "src/index.ts",
      exportPath: ".",
      format: ["esm", "commonjs"],
    },
  ],
  outDir: "dist",
};

await build(config);
```

<Note>
  If you call `build()` without arguments and no config file is found in the project root, Susee prints an error and exits with code `1`.
</Note>
