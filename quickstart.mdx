title: "Get started with Susee"
description: "Walk through installing Susee, generating a config file, and building your first dual-format library output with ESM, CommonJS, and declaration files."
---

This guide walks you through a complete Susee workflow — from installation to inspecting your compiled output. By the end you'll have a working `dist/` directory with ESM and CommonJS builds, sourcemaps, and TypeScript declaration files.

<Steps>
  <Step title="Install Susee">
    Add Susee as a dev dependency in your project:

    <CodeGroup>
      ```bash npm
      npm install -D susee
      ```
      ```bash pnpm
      pnpm add -D susee
      ```
      ```bash yarn
      yarn add -D susee
      ```
    </CodeGroup>
  </Step>

  <Step title="Initialize your config file">
    Run the init command from your project root:

    ```bash
    npx susee init
    ```

    Susee prompts: `Is TypeScript project(y/n)`. Press **Enter** or type `y` to create `susee.config.ts`. Type `n` for a JavaScript config (`.js` or `.mjs` depending on your `package.json` `type` field).

    The generated `susee.config.ts` looks like this:

    ```typescript susee.config.ts
    import type { SuSeeConfig } from "susee";

    const config: SuSeeConfig = {
      // Array of entry point objects.
      // ----------------------------
      entryPoints: [
        // You can add more entry points for different export paths.
        // NOTE: duplicate export paths are not allowed.
        // --------------------------------------------
        {
          // (required) Entry file path.
          entry: "src/index.ts", // replace with your entry file
          // (required) Export path for this entry.
          exportPath: ".", // "." stands for the main export path and can be set to "./foo", "./bar", etc.
          // (optional) Output module formats ["commonjs"] or ["esm", "commonjs"], default: ["esm"].
          // Uncomment the following line to edit.
          //format: ["esm"],
          // (optional) Rename duplicate declarations, default: true.
          // Uncomment the following line to edit.
          //renameDuplicates: true,
          // (optional) Custom tsconfig.json path, default: undefined.
          // Uncomment the following line to edit.
          //tsconfigFilePath: undefined,
          // (optional) Array of susee plugins, default: [].
          // Uncomment the following line to edit.
          //plugins: [],
          // (optional) Warning messages, if it true and warning message exist(1), default: false.
          // Uncomment the following line to edit.
          //warning: false,
        },
      ],
      // NOTE: the following options apply to all entry points.
      // ----------------------------------------------------------
      // (optional) Output directory, default: dist.
      // Uncomment the following line to edit.
      //outDir: "dist",
      // (optional) Allow susee to update your package.json, default: false.
      // Uncomment the following line to edit.
      //allowUpdatePackageJson: false,
    };

    export default config;
    ```
  </Step>

  <Step title="Edit your entry point">
    Update `entry` to point to your actual source file and set the output formats you need:

    ```typescript susee.config.ts
    import type { SuSeeConfig } from "susee";

    const config: SuSeeConfig = {
      entryPoints: [
        {
          entry: "src/index.ts",
          exportPath: ".",
          format: ["esm", "commonjs"],
        },
      ],
      outDir: "dist",
    };

    export default config;
    ```
  </Step>

  <Step title="Build your library">
    Run Susee from your project root:

    ```bash
    npx susee
    ```

    Susee clears the output directory, then compiles and bundles each entry point. Build time is printed to stdout on completion.
  </Step>

  <Step title="Inspect the output">
    For `entry: "src/index.ts"` with `format: ["esm", "commonjs"]` and `outDir: "dist"`, you'll find:

    | File | Description |
    |---|---|
    | `dist/index.mjs` | ESM bundle |
    | `dist/index.mjs.map` | ESM sourcemap |
    | `dist/index.d.mts` | ESM type declarations |
    | `dist/index.cjs` | CommonJS bundle |
    | `dist/index.cjs.map` | CommonJS sourcemap |
    | `dist/index.d.cts` | CommonJS type declarations |

    <Note>
      Declaration files (`.d.mts` / `.d.cts`) are only emitted when the TypeScript compiler produces them. Ensure `declaration: true` is set in your `tsconfig.json`.
    </Note>
  </Step>
</Steps>

## One-off builds without a config file

For quick builds, use `susee build` directly — no config file needed:

```bash
npx susee build src/index.ts --outdir dist
npx susee build src/index.ts --format commonjs --outdir dist
npx susee build --entry src/index.ts --format esm --minify
```

<Tip>
  See the [CLI build reference](/reference/cli-build) for all available flags and their defaults.
</Tip>
